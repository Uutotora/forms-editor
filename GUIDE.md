# Инструкция по работе с редактором вопросов

---

## Что это такое

Редактор вопросов — инструмент для просмотра и редактирования анкеты Росстат. Все изменения сохраняются напрямую в Google Таблицу. Никаких файлов, никаких выгрузок — всё работает онлайн.

---

## Интерфейс

```
┌─────────────────┬────────────────────────────────────────────┐
│                 │  [Undo] [Redo]  [Синхронизировать] [Сохранить] │
│  Список         ├────────────────────────────────────────────┤
│  вопросов       │                                            │
│                 │         Редактор вопроса                   │
│  Поиск...       │                                            │
│                 │                                            │
│  Вопрос 1       │                                            │
│  Вопрос 2  •    │                                            │
│  Вопрос 3       │                                            │
└─────────────────┴────────────────────────────────────────────┘
```

- **Левая панель** — список всех вопросов, поиск по номеру
- **Верхняя панель** — название текущего вопроса, кнопки управления
- **Центральная область** — форма редактирования

---

## Навигация по вопросам

Кликните на любой вопрос в левой панели — он откроется в редакторе.

В поле поиска можно вводить номер вопроса (например, `11.3` или `37`) — список сразу отфильтруется.

### Индикаторы рядом с вопросом

| Индикатор | Значение |
|---|---|
| Зелёная точка | Вопрос был сохранён в текущей сессии |
| Жёлтая точка | Есть несохранённые изменения (вы ушли без сохранения) |

---

## Редактирование вопроса

Каждый вопрос состоит из четырёх блоков.

### 1. Статус утверждения

Четыре кнопки: **Росстат**, **ДЭПР**, **НТУ**, **ДИТ**.

- Кликните на кнопку — статус переключится с "Нет" на "Да" и обратно
- Зелёный цвет — утверждено, серый — не утверждено
- Жёлтый цвет кнопки — значение изменено, но ещё не сохранено

При сохранении цвет ячеек в таблице обновится автоматически (зелёный / красный фон).

### 2. Карточка вопроса

Основные поля вопроса.

| Поле | Редактируется |
|---|---|
| ID вопроса | Нет (только просмотр) |
| Аббревиатура | Нет (только просмотр) |
| Тип заполнения | Нет (только просмотр) |
| Предусловие | Нет (только просмотр) |
| Текст вопроса | **Да** |
| Текст справки | **Да** |

Поля только для просмотра отображаются серым фоном — их нельзя изменить через редактор, только напрямую в таблице.

Если поле изменено и ещё не сохранено — оно подсвечивается жёлтым с надписью "Изменено · не сохранено".

### 3. Контроли

Таблица контролей — только для просмотра. Данные берутся из таблицы. Если контролей нет — отображается "Нет данных".

Чтобы увидеть содержимое — нажмите на заголовок блока, он раскроется.

### 4. Варианты ответов

Каждый вариант ответа показан карточкой. Содержит:

- **Мета-информацию** (chips сверху) — номер, аббревиатура и прочее. Только просмотр.
  - Синие chips: **Тип ответа**, **Тип варианта ответа**, **Переход на id** — ключевые поля
- **Текст заголовка ответа** — редактируется
- **Текст подсказки ответа** — редактируется

Чтобы увидеть варианты — нажмите на заголовок блока, он раскроется.

## Сохранение изменений

### Кнопка "Сохранить" (или Ctrl+S / Cmd+S)

Сохраняет изменения текущего вопроса в Google Таблицу.

- Кнопка активна только если есть несохранённые изменения (горит синим)
- После сохранения блоки, в которых были изменения, на секунду подсвечиваются зелёным с надписью "Сохранено"
- В таблицу записываются **только изменённые ячейки** — остальные не трогаются

> Сохранение работает только для текущего открытого вопроса. Чтобы сохранить другой вопрос — перейдите на него.

---

## Отмена изменений (Undo / Redo)

Кнопки со стрелками в верхней панели (или Ctrl+Z / Ctrl+Shift+Z).

- **Undo** — отменить последнее действие
- **Redo** — вернуть отменённое

История сохраняется пока вы не покинете вопрос или не перезагрузите страницу.

---

## Что происходит при переходе на другой вопрос с несохранёнными изменениями

Появится диалог с тремя вариантами:

| Вариант | Что произойдёт |
|---|---|
| **Сохранить и перейти** | Изменения запишутся в таблицу, затем откроется новый вопрос |
| **Перейти без сохранения** | Изменения останутся в памяти (жёлтая точка в сайдбаре). При возврате они восстановятся |
| **Отмена** | Остаться на текущем вопросе |

> "Перейти без сохранения" — не потеря данных. Изменения хранятся в памяти браузера до закрытия вкладки. При возврате к вопросу они восстановятся автоматически.

## Синхронизация с таблицей

### Когда нужна синхронизация

Синхронизация нужна если кто-то **напрямую изменил данные в Google Таблице** — например, добавил новый вопрос, изменил контроли или варианты ответов. Редактор кэширует данные и без синхронизации не видит внешних изменений.

> Если вы сами редактируете только через этот редактор — синхронизация нужна реже.

### Как синхронизировать

Нажмите кнопку **"Синхронизировать"** в верхней панели.

Что произойдёт:
1. Сервер сбросит кэш и загрузит актуальные данные из таблицы
2. Список вопросов обновится (новые вопросы появятся, удалённые исчезнут)
3. Текущий открытый вопрос перезагрузится с актуальными данными
4. Все "сохранённые" метки (зелёные точки) сбросятся — таблица теперь источник истины

### Предупреждение перед синхронизацией

Если у вас есть **несохранённые изменения**, перед синхронизацией появится предупреждение. Синхронизация перезапишет несохранённые данные данными из таблицы.

- Сначала сохраните изменения, потом синхронизируйте
- Или подтвердите синхронизацию — несохранённые изменения будут потеряны

## Частые сценарии

### Я изменил текст вопроса — как сохранить?

1. Внесите изменения в поле "Текст вопроса"
2. Нажмите **Сохранить** или **Ctrl+S**
3. Блок "Карточка вопроса" мигнёт зелёным — изменения записаны в таблицу

### Коллега изменил таблицу напрямую — как увидеть изменения?

Нажмите **Синхронизировать**. Данные обновятся.

### Я случайно что-то изменил — как вернуть?

Нажмите **Ctrl+Z** (или кнопку Undo) — изменение откатится. Можно нажимать несколько раз.

### Хочу перейти на другой вопрос и вернуться потом

Просто перейдите — в диалоге выберите "Перейти без сохранения". Жёлтая точка покажет что изменения ждут. При возврате всё восстановится.

### Как найти нужный вопрос?

Введите номер в поле поиска слева (например, `11.3` или `37`). Список отфильтруется по номеру.

## Горячие клавиши

| Клавиша | Действие |
|---|---|
| Ctrl+S / Cmd+S | Сохранить текущий вопрос |
| Ctrl+Z / Cmd+Z | Отменить последнее действие |
| Ctrl+Shift+Z / Cmd+Shift+Z | Повторить отменённое |
| Ctrl+Y | Повторить отменённое (альтернатива) |
